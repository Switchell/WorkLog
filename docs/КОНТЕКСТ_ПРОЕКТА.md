# WorkLog (AXIS ERP) — контекст проекта для разработки / ИИ

Документ фиксирует **как устроен проект**, **тонкости эксплуатации** и **что уже менялось**. Без секретов: пароли только в `.env` и в личных заметках владельца.

**См. также:** [СОСТОЯНИЕ_И_ROADMAP.md](./СОСТОЯНИЕ_И_ROADMAP.md) — аудит, пробелы и план выхода на более высокий уровень зрелости.

---

## 1. Назначeние

- **Админка (Streamlit):** учёт часов, проектов, ставок, импорт Excel, PDF, бэкапы PostgreSQL, отчёты в Telegram.
- **Клиентский портал (Streamlit):** просмотр агрегатов по проектам/выручке (логин по паролям клиентов).
- **Grafana:** дашборды поверх той же БД `workday_db`.
- **Nginx:** единая точка входа `:80`/`:443`; Streamlit админка — `/`, клиент — `/client/`.

## 3. Docker: сервисы и порты

| Сервис   | Контейнер    | Порты на хосте | Примечание |
|----------|--------------|----------------|------------|
| postgres | `axis_db`    | `5432:5432`    | том `postgres_data` — **данные учёта здесь** |
| admin    | `axis_admin` | `8502:8501` + внутренний 8501 | основной UI также через nginx `/` |
| client   | `axis_client`| `8503:8501` (опционально) + `expose 8501` | в проде обычно через nginx `/client/`; **8503** — прямой Streamlit для отладки |
| nginx    | `axis_nginx` | `80`, `443`    | прокси на admin/client |
| grafana  | `axis_grafana` | `3001:3000`  | том `grafana_data` + provisioning |
| certbot  | `axis_certbot` | —          | обновление сертификатов |

**Важно:** у `admin`/`client` в compose долгое время был только `expose` — в Docker Desktop **не было “ссылки” на порт**; для админки добавлен проброс **8502:8501** для прямого доступа с хоста.

**Клиентский Streamlit** за nginx: путь `/client/` должен соответствовать настройке Streamlit `baseUrlPath=client` **и** `location /client/` в nginx — иначе `_stcore` и health 404. (Исторически ломалось без этого; откатывали по просьбе пользователя — актуальный `nginx.conf` и `Dockerfile_client` смотреть в репо.)

**DB_HOST в Docker:** у сервисов `admin` и `client` в compose задано `environment: DB_HOST: db`, чтобы даже при `DB_HOST=axis_db` в `.env` приложение подключалось по имени сервиса **`db`**, а не по `container_name`.

## 4. База данных

- Имя БД в compose по умолчанию: `workday_db`, пользователь `user` (пароль в `docker-compose` у сервиса `db` должен совпадать с тем, что в `grafana/.../datasources.yml`, если не вынесено в env).
- Таблицы: `work_logs` (`Date`, `Sotrudnik`, `Proect`, `Time`), `employee_rates` (`Sotrudnik`, `Rate`, `Client_Rate`).  
  Поле проекта в БД: **`Proect`** (опечатка в схеме — везде так, не «Project»).
- Миграции «в лоб» в `app.py` при `init_connection()`: `CREATE TABLE IF NOT EXISTS`, снятие старого unique при необходимости.

## 5. Переменные окружения (обзор)

См. `.env.example`. Ключевые группы:

- **DB_*** — хост из контейнера обычно `db`, порт `5432`.
- **TG_TOKEN**, **TG_ADMIN_ID** — Telegram; **без VPN/доступа к api.telegram.org** запросы таймаутятся (это норма для части сетей).
- **TG_CONNECT_TIMEOUT**, **TG_READ_TIMEOUT**, **TG_SEND_RETRIES** — тонкая настройка отправки.
- **AXIS_CLIENT_KWORK_PASSWORD**, **AXIS_CLIENT_FREELANCE_PASSWORD** — переопределение паролей клиентского портала (иначе дефолты из кода).
- **DB_POOL_SIZE**, **DB_MAX_OVERFLOW** — пул SQLAlchemy в `worklog_db.py`.

## 6. Grafana — тонкости

1. **Обязателен том** `grafana_data`, иначе при любом пересоздании контейнера теряются дашборды/пароль админа в UI.
2. **Grafana 12:** тип источника в JSON панелей — **`grafana-postgresql-datasource`**, не `postgres`. Иначе панели «пустые» при рабочей БД. `uid` источника можно оставить `postgres`.
3. Provisioning: хост PostgreSQL из контейнера Grafana — **`db:5432`** (имя **сервиса** compose), не `axis_db`.
4. Каноничный дашборд в репо: `docker/grafana/dashboards/axis-dashboard.json` (копия/синхрон с `docs/grafana/dashboard.json`).

---

## 7. Telegram

- Реализация в **`worklog_telegram.py`**: прямой `requests.post` на Bot API (не pyTelegramBotAPI), ретраи, отдельные исключения сети vs API.
- В **`worklog_telegram_reports`** при сетевой ошибке показывается подсказка про VPN; сбой TG **не** отменяет сохранение данных в БД.

## 8. История значимых изменений (хронология темы с Cursor)

- Исправление **nginx + Streamlit** для `/client/` (baseUrlPath + proxy_pass) — откат по запросу пользователя; проверять текущее состояние файлов.
- **`docker container prune`** — удаляет **остановленные** контейнеры; тома БД не трогает. Поднимались отдельно `n8n-gemini`, `logos_system`; у `logos_system` порты на хосте смещены (5433, 5679), чтобы не конфликтовать с WorkLog.
- **Админка:** порт **8502** для прямого доступа к Streamlit.
- **Grafana:** том + provisioning + исправление типа datasource под v12.
- **Код:** модули `worklog_db`, `worklog_telegram`; пакетный UPSERT в импорте; кэш/пароли в `app_client`; `.env.example`; правки Plotly (`use_container_width`) под Streamlit 1.50; `get_logs` без зависимости от `USER_ROLE` внутри кэша.

## 10. Быстрая диагностика

| Симптом | Куда смотреть |
|--------|----------------|
| Пустой Grafana | Тип datasource в JSON панелей; `db:5432`; пароль; логи `axis_grafana` |
| TG timeout | VPN; `TG_*` таймауты; логи `axis_errors.log` / Streamlit |
| Клиент 404 под `/client` | `nginx.conf`, `--server.baseUrlPath` у client, перезапуск client+nginx |
| Нет данных в админке | `.env` DB_*; том `postgres_data`; контейнер `axis_db` |
